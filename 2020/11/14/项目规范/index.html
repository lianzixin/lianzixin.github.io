项目文件命名规范

  • 常量文件，命名使用 XXXConsist
  • 枚举文件，命名使用 XXXEnum
  • 控制器文件，命名使用 XXXController
  • 数据访问层文件，命名使用 XXXRepository
  • 请求对象文件，命名使用 XXXReq
  • 响应对象文件，命名使用 XXXRsp
  • 接口文件，命名使用 XXXService
  • 接口实现文件，命名使用 XXXServiceImpl
  • 工具文件，命名使用 XXXUtil

GitHub Commit 规范

Git 每次提交代码，都按照以下规范书写 Commit message，否则不允许提交

Commit message 的格式
其中 Header 是必须的，Body、Footer 可以省略

Header 部分包括三个字段：type,scope,subject
type：用于说明 commit 的类别：

  • feat：新功能
  • fix：修复 bug
  • docs：文档
  • style：格式
  • refactor：重构
  • test：增加测试
  • chore：构建过程或辅助工具的变动

scope：用于说明 commit 影响的范围
subject：commit 目的的简短描述

Body 部分是对本次 commit 的详细描述

  • 使用第一人称现在时
  • 说明代码变动的动机，以及与以前行为的对比

Footer

  • 说明不兼容的变动
  • 关闭 Issue

Revert 当前 commit 用于撤销以前的 commit，则必须以 revert: 开头

Commitizen
撰写合格 Commit message 的工具
安装命令：npm install -g commitizen
支持 Angular 规范：commitizen init cz-convertional-changelog --save --save-exact
使用命令：git cz

validate-commit-msg
用于检查 Node 项目的 Commit message 是否符合格式
