如何创建 React 项目

2020-12-01 how to

创建 Typescript 项目

  1. 创建文件夹：mkdir app
  2. 进入 app 文件夹，初始化 npm ：npm init //根据提示生成 package.json 文件
  3. 安装 typescript/tslint 和 NodeJs 的类型声明：npm install --save-dev typescript tslint @types/node // tslint 是代码规则检查工具，保持代码风格一致
  4. 配置 tsconfig.json：./node_modules/.bin/tsc --init //ts 配置信息
  5. 配置 tslint.json：./node_modules/.bin/tslint --init //配置规则信息
  6. 在 src 目录创建 index.ts 作为入口文件：/src/index.ts // 需要在 tsconfig 配置
  7. 编译：tsc //将会生成对应的 js 文件，需要在 tsconfig 配置输出目录
  8. 运行：node ./dist/index.js //运行 js 文件

创建 TypeScript-React app

  1. 初始化项目：npx create-react-app <name> --template typescript
  2. 安装 eslint：npm install eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
  3. eslint 初始化：eslint --init
  4. 安装 prettier：npm install prettier eslint-config-prettier eslint-plugin-prettier
  5. 安装 editorconfig：npm install -g editorconfig

EditorConfig
VSCode 使用 Generate .editorconfig 命令
EditorConfig 可以帮助开发者在不同的编辑器和 IDE 之间定义和维护一致的代码风格。

ESLint
ESLint 是一个 Javascript Linter，帮助我们规范代码质量，提高团队开发效率。

Prettier
prettier 是一个能够统一团队编码风格的工具
增加 prettier 配置文件，根目录创建 .prettierrc.yaml

.eslintrc 推荐使用

VsCode 开启保存修复功能
需要修改 setting.json 文件，安装 eslint、prettier、editorconfig 插件

React 使用环境变量

在 .env 中添加开发环境变量，对于环境变量必须使用 REACT_APP_ 开头

  • .env：默认环境
  • .env.local：本地覆盖
  • .env.development，.env.test，.env.production：设置特定环境

使用自定义环境变量

  1. 创建 .env.[env] 的文件，设置对应变量
  2. 安装 env-cmd：npm install env-cmd --save
  3. package.json 添加脚本："build:staging：'env-cmd .env.staging npm run build'"
